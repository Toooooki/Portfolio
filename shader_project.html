Pixel Art Shader for Unity

URP Fullscreen Shaderを用いた、3Dモデルのドット絵風リアルタイムレンダリング

Technologies Used

Unity (URP) HLSL Shader Graph Post-Processing

Project Info

Status: In Development
Base: Unity-URP-Outlines
Goal: VRoid Game Integration

Overview / 背景

個人開発でリッチな3Dゲームを作りたいが、3Dモデル作成のコストはかけられないという課題がありました。 そこで、高品質なキャラクターモデルを容易に作成できる VRoid Studio を採用しましたが、そのまま使うと「よくあるVRoid製のゲーム」として埋もれてしまいます。

また当初は2Dドット絵のゲーム開発も検討しましたが、キャラクターのアニメーション全パターンを描き下ろす工数が膨大で断念しました。 そこで、3Dモデルの手軽さとドット絵独自の味を両立させる解決策として、このシェーダー開発に着手しました。

Technical Evolution / 技術的アプローチの変遷

Phase 1: RenderTextureアプローチ（初期案）

開発当初は、カメラの描画内容を低解像度の RenderTexture を使用する方法を試みました。 しかし、この手法ではジャギーのコントロールが難しく、単に解像度を下げただけの荒い映像になりがちで、意図したドット絵らしさが出せませんでした。

初期プロトタイプ：RenderTextureによる低解像度化

Phase 2: URP Fullscreen Shaderへの移行（現在）

より高度な制御を行うため、URPのポストプロセス（Fullscreen Shader）による実装に切り替えました。 ベースとしてオープンソースの Unity-URP-Outlines を採用し、独自の改造を施しています。

Fullscreen Shaderの改造画面

主な改造点：
元のシェーダーはシンプルな輪郭線描画のみでしたが、法線（Normal）情報を解析し、 凸（山折り）と凹（谷折り）で輪郭線の色を変える処理を追加し凹凸の判別がつきやすいようにしました。

Current Challenges & Future / 課題と展望

現在の課題：
オブジェクト同士が重なった際（深度値が近い場合）、輪郭線の描画が干渉してしまう問題が発生しています。 現在、Depth BufferとNormal Bufferの比較ロジックを見直し、重なり順を正しく判定できるよう修正中です。

今後の展望：
この問題が解決次第、現在開発を一時停止しているVRoidベースの3Dアクションゲームに本シェーダーを適用し、 独自のビジュアルスタイルを持つ作品として完成させる予定です。
